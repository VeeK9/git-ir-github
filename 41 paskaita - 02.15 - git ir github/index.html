Git

Git'as yra populiariausia kodo versijavimo programa.

Git komandos

  • init - sukuria git'o repozitorija dabartiniame aplankale
  • status - nurodo informacija apie lokalios git'o repozitorijos statusa
  • add - prideda nurodytus failus prie commitui paruostu (stage'ina)
  • rm --cached - isima nurodytus failus is stage'inimo zonos
  • commit -m "zinute" - sukuria kodo versija is stage'intu failu ir prideda zinute apie pakitimus (ne ilgesnes nei 50 simboliu) ir turi buti aiski.
  • restore - grazina nurodytus failus i buvusio commito busena (panaikina naujus pakitimus padarytus po paskutinio commit'o)
  • push - is lokalios repozitorijos nusiuncia commit'us i isorine repozitorija
  • pull - is isorines repozitorijos atsiuncia commit'us i lokalia repozitorija
  • clone - klonuoja nurodyta repozitorija is isorines repozitorijos
  • log - isveda commit'u sarasa
  • checkout commitID | branchName - pereina i nurodyta kodo versija ARBA i kita saka
  • branch naujosSakosPavadinimas - sukuria nauja saka

Branch protection rules

Vienas svarbiausiu saku apsaugos taisykliu yra "Pull Request" (bandymas vienos sakos commit'us sukelti i kita saka).
Pull Request dar galima suteikti "Reiquire approvals", kuris reikalauja, kad nurodytas kiekis kitu projekto dalyviu patvirtintu Pull Request'a.
